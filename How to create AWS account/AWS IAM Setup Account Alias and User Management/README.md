# AWS IAM Setup: Account Alias and User Management

**Creating and Configuring IAM in AWS**

Start by opening the IAM dashboard from the AWS Management Console. Create an account alias to replace the long numeric AWS account ID with a readable name, this makes the sign-in URL easier to remember. Then create an IAM user by defining a username and enabling console access with either an autogenerated or custom password.

Assign permissions by attaching the appropriate policy. For initial setup or learning, AdministratorAccess provides full access. Add optional tags to organize and identify the user. Review the details, create the user, and securely save the login credentials and console sign-in URL.

Log out of the root account and sign in with the new IAM user to confirm everything works. Enable Multi-Factor Authentication (MFA) on the IAM user for added security. From now on, use the root account only for billing and account-level settings and perform all daily AWS activities with the IAM user.

**STEP ONE**

Open IAM Dashboard:

- Sign in to the **AWS Management Console**
- In the search bar, type **IAM**
- Click **Identity and Access Management (IAM)**

![image.png](images/image.png)

**STEP TWO**

Add an Account Alias (Change Account ID Display)

- On the IAM dashboard, locate **AWS Account** (top right panel)
- Click **Create** next to **Account Alias**
- Enter a unique alias name (e.g. your-cloud-account)
- Click **Create alias**

![image.png](images/image%201.png)

**STEP THREE**

Create an IAM User:

- In the IAM left menu, click **Users**
- Click **Create user**
- Enter a **username** (e.g. admin-user)
- Select access type:
    - **AWS Management Console access**
- Choose **Custom password** or **Autogenerated password**
- (Optional but recommended) Check **User must create a new password at next sign-in**
- Click **Next**

![image.png](images/image%202.png)

![image.png](images/image%203.png)

**STEP FOUR**

Set Permissions for the User:

- Search for **AdministratorAccess**
- Select it
- Click **Next**

![image.png](images/image%204.png)

**STEP FIVE**

Review and Create User:

- Review username and permissions
- Click **Create user**

![image.png](images/image%205.png)

**STEP SIX**

Save Login Details:

- Copy the **Console sign-in URL** (this includes your alias)
- Save the **username and password** securely

You will not see the password again if autogenerated.

![image.png](images/image%206.png)

**STEP SEVEN**

Log Out and Test IAM User:

- Log out of the root account
- Log in using the **IAM user credentials**
- Confirm successful login

![image.png](images/image%207.png)

**STEP EIGHT**

Secure IAM User with MFA (Recommended):

- Go to **IAM → Users**
- Select the user
- Open **Security credentials**
- Assign **MFA device**

![image.png](images/image%208.png)

**STEP NINE**

Avoid Using Root Account:

- Use the **IAM user** for all daily tasks
- Keep the **root account only for billing and account-level settings**
